mtf-classifier

The following capabilities are provided by mtf-classifier:

  • Produce subset XML Schema containing relevant XML-MTF data components.
  • Componentize XML-MTF data components into smaller conceptual components.
  • Organize XML-MTF data components into conceptually-related taxonomic classes.
  • Reuse with future XML-MTF releases.

Produce Subset XML Schema

XML Schema are produced containing a subset of XML-MTF data components. The subset XML-MTF data components are selected by XSL instructions that read and process XML-MTF XML Schema. The instructions obtain relevant data components implemented by the TSOA-MTF data format.

The following diagram dipicts the process for producing a subset of Location related data components:

    /-----------------------------------------\    /----------------------------\
    |  USMTF Mil-Std-6040 XML-MTF XML Schema  |    |  [ classifyLocation.xsl ]  |
    \-----------------------------------------/    \----------------------------/
                                         \              /
                                          |             |
                                          V             V
                                        /-------------------\
                                       /   XSL Transformer   \
                                       \---------------------/
                                                  |
                                                  V
                                      /-----------------------\
                                      |  [ mtfLocation.xsd ]  |
                                      \-----------------------/

Each data component is an unaltered representation published in XML-MTF XML Schema. However, the published XML-MTF data components are further componentized for the subset schema to facilitate their implementation.

Componentize XML-MTF

Further componentization of XML-MTF data components is performed by XSL instructions that match data component names with match criteria. Match criteria are configurable by modifying file [ mtfClassGraph.xml ].

Increased componentization facilitates the replacement of embedded XML-MTF data components with NIEM data components. For example, if a NIEM component for Location replaces an XML-MTF Location component embedded in an XML-MTF EventData component, the XML-MTF EventData component requires componentization. Componentization of its embedded XML-MTF data components facilitates the exclusion of the XML-MTF Location component and the inclusion of remaining XML-MTF data components in the TSOA-MTF data format.

The following diagram conceptually depicts the previously described example:

    /---------------------------------------------------------\
    |  mtf:EventData  +--  mtf:StageOfIntelligenceEvaluation  |
    |                 |                                       |
    |                 +--  mtf:LocationLatLongDeciseconds     |
    |                 |                                       |
    |                 +--  mtf:EventType                      |
    |                 |                                       |
    |                 +--  mtf:...                            |
    \---------------------------------------------------------/

is componentized as

    /-----------------\
    |  mtf:EventData  |
    \-----------------/
    
    /-------------------------------------\
    |  mtf:StageOfIntelligenceEvaluation  |
    \-------------------------------------/
    
    /----------------------------------\
    |  mtf:LocationLatLongDeciseconds  |
    \----------------------------------/
    
    /-----------------\
    |  mtf:EventType  |
    \-----------------/
    
    /-----------\
    |  mtf:...  |
    \-----------/

so that

    /-------------------------------------------------------------------------------------\
    |  tsoa:Event  +--  niem:Location                                                     |
    |              |                                                                      |
    |              +--  tsoa:MtfEventDataAdapter  +--  mtf:StageOfIntelligenceEvaluation  |
    |                                             |                                       |
    |                                             +--  mtf:EventType                      |
    |                                             |                                       |
    |                                             +--  mtf:...                            |
    \-------------------------------------------------------------------------------------/

The Adapter Pattern as proscribed by the NIEM Naming and Design Rules (NDR) is used to reference unaltered XML-MTF data components, which are NIEM non-conforming.

The componentization of XML-MTF data components produces conceptually related data components that can be organized into taxonimic classes.

Organize XML-MTF Components into Classes

Conceptually related data components are organized into taxonimic classes represented by XML Schema. For example, data components related to the description of a geospatial location are found in the file [ mtfLocation.xsd ].

The following are sample classes represented by XML Schema:

    |----------|    |------------|    |------------|    |-----------|    |--------------|
    |  Entity  |    |  Activity  |    |  Location  |    |  Measure  |    |  Identifier  |
    |----------|    |------------|    |------------|    |-----------|    |--------------|

Organizion by conceptual class rather than function alone is a common practice for improved reuseability. For example, finding a data component for reuse that is associated to a conceptual class is more intuitive than finding it by the function proscribed in the USMTF specification. In other words, organization by conceptual class rather than function improves useability by abstracting a user from the details proscribed in the USMTF specification.

The file [ mtfClassGraph.xml ] describes the data component classes.

Reuse with Future XML-MTF Releases

The mtf-classifier software component is reuseable for future releases of XML-MTF XML Schema. Simply process the new version of XML-MTF XML Schema once again to update the subset XML Schema. Again, the selection of XML-MTF data components is determined by the criteria set forth in the file [ mtfClassGraph.xml ].

With the subset XML Schema updated, the TSOA-MTF XML Schema should work without modification. Modification of the TSOA-MTF XML Schema may be required when, for example, new XML-MTF data components are introduced by the newer version of XML-MTF XML Schema.
